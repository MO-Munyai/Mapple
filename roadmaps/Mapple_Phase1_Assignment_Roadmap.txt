Mapple v0.2.x Phase 1: Assignment Support

PROBLEM STATEMENT

Current state:

-   The lexer already supports ‘=’.
-   The parser recognizes assignment syntax but returns placeholder
    strings.
-   No real AssignmentNode exists in the AST.
-   Semantic analysis does not validate assignments.
-   Code generation ignores assignments.

As a result, code such as:

    let int age = 20;
    age = 25;

cannot be processed end-to-end by the compiler.

GOAL

Implement full assignment support across:

1.  AST
2.  Parser
3.  Semantic Analysis
4.  Code Generation
5.  Testing
6.  Documentation

Definition of success:

    let int age = 20;
    age = age + 1;
    print(age.str);

should output:

    21

while invalid assignments should be rejected.

==================================================================
DEFINITION OF DONE
==================================================================

Phase 1 is complete when:

1.  This compiles and runs:

    let int age = 20; age = age + 1; print(age.str);

Output:

    21

2.  This fails:

    age = 10;

because age was never declared.

3.  This fails:

    let int age = 10; age = “hello”;

because assignment types do not match.

4.  Assignment is represented by a real AssignmentNode.

5.  Assignment is parsed, semantically validated, generated into Python,
    and documented.

At that point assignment support is considered fully implemented
end-to-end.
